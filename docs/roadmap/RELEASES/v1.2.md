# v1.2 — Connected Executive

**Status:** Planned  
**Theme:** Google Workspace integration — bring the executive's external context into MyBoss360  
**Target:** Q3 2026

---

## Objective

An executive's most critical context lives in three places: their calendar, their email, and their contacts. Today, MyBoss360 cannot see any of it. The executive must manually copy meeting notes, paste email threads, and enter contact data — a tax on their time that contradicts the platform's core purpose.

v1.2 connects MyBoss360 to Google Workspace. Calendar events flow into the Executive Agenda. Email threads are ingested and summarized. Contacts enrich the CRM. Documents from Google Drive feed the Knowledge Engine.

After v1.2, the executive's MyBoss360 instance knows what is on their agenda today, what has been discussed with key contacts, and what documents exist — without any manual input.

## Features

### Google Calendar Integration

**Goal:** Surface the executive's schedule as a first-class data source.

| Capability | Description |
|---|---|
| Event sync | Pull calendar events into `calendar_events` table (incremental sync via webhooks) |
| Executive Agenda view | Unified daily/weekly agenda: meetings + tasks + priorities in one view |
| Meeting context | Before each meeting, surface relevant contacts, deals, documents, and decisions |
| Meeting notes | After a meeting, create a Knowledge Engine document (type: `meeting_notes`) |
| Conflict detection | Flag scheduling conflicts and suggest resolution |

**Sync strategy:**
- Primary: Google Calendar push notifications (webhooks) for real-time updates
- Fallback: polling every 15 minutes if webhook registration fails

### Google Gmail Integration

**Goal:** Ingest and summarize email communication for executive context.

| Capability | Description |
|---|---|
| Thread ingestion | Sync email threads involving key contacts into the knowledge layer |
| Thread summarization | AI-generated summary of each thread on ingestion |
| Contact linking | Associate email threads with CRM contacts automatically |
| Action item extraction | Identify and surface commitments, follow-ups, and decisions from email |
| Unread digest | Daily AI-generated summary of unread emails requiring executive attention |

**Privacy:** Only threads where the executive is a participant are ingested. No bulk inbox archiving. Workspace admins can configure which label scopes are synced.

### Google Contacts Integration

**Goal:** Enrich CRM contacts with Google Contacts data.

| Capability | Description |
|---|---|
| Contact enrichment | Fill missing fields (phone, title, company) from Google Contacts |
| Bidirectional sync | New CRM contacts optionally created in Google Contacts |
| Deduplication | Merge candidates surfaced for executive review (no auto-merge) |

### Google Drive Integration

**Goal:** Ingest documents from Google Drive into the Knowledge Engine.

| Capability | Description |
|---|---|
| Drive folder sync | Connect a Drive folder; all documents are parsed, chunked, and stored |
| Supported types | Google Docs (exported as text), PDF, DOCX |
| Incremental sync | Re-ingests only modified files (based on Drive `modifiedTime`) |
| Source tracking | Documents linked to `knowledge_sources` record (type: `integration`, subtype: `google_drive`) |

### Executive Agenda

**Goal:** A single view that tells the executive exactly what today and this week look like.

The Executive Agenda combines:
- Google Calendar events (with meeting context cards)
- Tasks due today/this week
- Deals requiring action (follow-up overdue)
- Knowledge items recently updated
- AI-generated daily briefing (what matters most today)

## Architecture

### OAuth Credential Store

Each workspace stores its own Google OAuth tokens, encrypted via Supabase Vault:

```sql
CREATE TABLE workspace_integrations (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id    UUID REFERENCES workspaces(id),
    provider        TEXT NOT NULL,  -- 'google'
    access_token    TEXT,           -- encrypted via Vault
    refresh_token   TEXT,           -- encrypted via Vault
    token_expires_at TIMESTAMPTZ,
    scopes          TEXT[],
    status          TEXT DEFAULT 'active',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW()
);
```

### Sync Engine

```
Google Webhook / Polling
    │
    ▼
app/api/integrations/google/webhook/route.ts
    │
    ├─ Verify webhook signature
    ├─ Identify resource type (calendar | gmail | drive)
    └─ Enqueue sync job
            │
            ▼
        SyncWorker
            ├─ Fetch changed resources from Google API
            ├─ Upsert into local tables
            └─ Trigger pipeline (Gmail/Drive → Knowledge Engine)
```

## Success Criteria

| Criterion | Target |
|---|---|
| Google OAuth connects in < 3 clicks | Standard OAuth 2.0 PKCE flow |
| Calendar events sync within 30 seconds | Webhook registration + delivery |
| Executive Agenda loads in < 1 second | Server-side rendered, cached |
| Email thread ingestion completes in < 5 seconds | Pipeline SLA |
| Drive document sync completes in < 30 seconds per document | Parse + chunk + store |
| No email content stored in plain text | Encryption at rest via Supabase |
| Revoking Google access removes all synced data | Full workspace cleanup on disconnect |
| Lint clean | 0 errors |
| Production build passes | 0 errors |
| Test coverage ≥ 70% on new services | Vitest + V8 |

- Supabase Vault for encrypted credential storage
- `workspace_integrations` table migration
